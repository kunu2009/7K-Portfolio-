## What is React? {#what-is-react}

**React** is a JavaScript **library** for building user interfaces, created by Facebook (Meta) in 2013.

### Core Characteristics

**1. Component-Based Architecture**
```jsx
// Simple React Component
function Welcome({ name }) {
  return <h1>Hello, {name}!</h1>;
}

export default function App() {
  return (
    <div>
      <Welcome name="Kunal" />
      <Welcome name="World" />
    </div>
  );
}
```

**2. Virtual DOM**
React uses a virtual DOM for efficient updates. When state changes, React:
1. Creates a virtual DOM tree
2. Compares it with the previous tree (diffing)
3. Updates only changed parts in real DOM

**Performance benefit:** 10-100x faster than direct DOM manipulation

**3. Client-Side Rendering (CSR)**
- JavaScript executes in the browser
- Initial HTML is nearly empty
- Content loads after JavaScript executes

**Initial HTML (React):**
```html
<!DOCTYPE html>
<html>
  <body>
    <div id="root"></div>
    <script src="/bundle.js"></script>
  </body>
</html>
```

### React Ecosystem

React is a library, not a framework. You need to add:
- **Routing:** React Router (2M+ weekly downloads)
- **State Management:** Redux, Zustand, Jotai
- **Data Fetching:** React Query, SWR, Axios
- **Forms:** React Hook Form, Formik
- **Styling:** Styled-components, Emotion, Tailwind CSS
- **Build Tool:** Vite, Create React App, Webpack

**Pros:**
‚úÖ Flexibility - choose your own tools
‚úÖ Large ecosystem (100,000+ npm packages)
‚úÖ Easier to learn basics

**Cons:**
‚ùå Decision fatigue
‚ùå Configuration overhead
‚ùå No standard way to do things

### When React Shines

**Best for:**
- Single Page Applications (SPAs)
- Internal dashboards and tools
- Applications behind authentication
- Mobile apps (React Native)
- Projects with existing React setup

**Popular React Apps:**
- Facebook
- Instagram
- WhatsApp Web
- Netflix (parts of UI)
- Airbnb

**Source:** [React Official Docs](https://react.dev/)

## What is Next.js? {#what-is-nextjs}

**Next.js** is a **React framework** for production, created by Vercel in 2016. It's React + batteries included.

### Core Characteristics

**1. Multiple Rendering Strategies**

**Server-Side Rendering (SSR):**
```jsx
// Next.js 15 Server Component (default)
async function ProductPage({ params }) {
  const product = await fetchProduct(params.id);
  
  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.description}</p>
    </div>
  );
}
```

HTML is generated on each request. Perfect for dynamic content.

**Static Site Generation (SSG):**
```jsx
// Generated at build time
export async function generateStaticParams() {
  const products = await fetchAllProducts();
  return products.map((p) => ({ id: p.id }));
}
```

HTML generated once at build time. Blazing fast!

**Incremental Static Regeneration (ISR):**
```jsx
// Revalidate every 60 seconds
export const revalidate = 60;

async function BlogPost({ params }) {
  const post = await fetchPost(params.slug);
  return <article>{post.content}</article>;
}
```

Static page that updates periodically. Best of both worlds.

**2. File-Based Routing**

## Key Differences Explained {#key-differences}

### 1. Rendering Methods

**React (Client-Side Rendering):**

```
Browser Request ‚Üí Empty HTML ‚Üí Download JS ‚Üí Execute JS ‚Üí Render Content
```

**Timeline:**
- HTML: 50ms
- JavaScript download: 500ms
- JavaScript execution: 300ms
- **Total Time to Interactive: ~850ms**

**Next.js (Server-Side Rendering):**

```
Browser Request ‚Üí Server Renders HTML ‚Üí Send Full HTML ‚Üí Hydrate
```

**Timeline:**
- Server render + HTML: 200ms
- JavaScript hydration: 150ms
- **Total Time to Interactive: ~350ms**

**Performance difference:** Next.js is **2-3x faster** for first contentful paint.

**Real benchmark (on 3G connection):**
| Metric | React (CSR) | Next.js (SSR) |
|--------|-------------|---------------|
| First Byte | 100ms | 200ms |
| First Contentful Paint | 1.8s | 0.9s |
| Time to Interactive | 3.2s | 1.2s |
| SEO Score | 65/100 | 95/100 |

**Source:** [Web.dev Case Studies](https://web.dev/case-studies/)

### 3. Data Fetching

**React (Client-Side):**

```jsx
function BlogPost({ id }) {
  const [post, setPost] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  
  useEffect(() => {
    fetch(`/api/posts/${id}`)
      .then(res => res.json())
      .then(data => {
        setPost(data);
        setLoading(false);
      })
      .catch(err => {
        setError(err);
        setLoading(false);
      });
  }, [id]);
  
  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;
  
  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}
```

**Issues:**
- ‚ùå Waterfall requests (wait for JS ‚Üí then fetch data)
- ‚ùå Loading spinners everywhere
- ‚ùå No data in initial HTML (bad for SEO)
- ‚ùå Repetitive error handling

**Next.js (Server-Side):**

```jsx
// Data fetching happens on server
async function BlogPost({ params }) {
  const post = await fetchPost(params.id);
  
  return (
    <article>
      <h1>{post.title}</h1>
      <p>{post.content}</p>
    </article>
  );
}
```

**Benefits:**
- ‚úÖ Data ready in initial HTML
- ‚úÖ No loading spinners
- ‚úÖ Better SEO
- ‚úÖ Built-in error handling
- ‚úÖ Automatic TypeScript types

**Performance comparison:**

```
React: HTML (50ms) ‚Üí JS (500ms) ‚Üí Fetch (300ms) = 850ms
Next.js: Fetch + Render (300ms) ‚Üí HTML (300ms) = 600ms
```

**Next.js is 30-40% faster** for data-heavy pages.

### Step-by-Step Migration

**Step 1: Setup Next.js**
```bash
npx create-next-app@latest my-next-app
```

Choose:
- TypeScript: Yes
- ESLint: Yes
- Tailwind: Yes (if you use it)
- App Router: Yes

---

**Step 2: Copy Components**
```
src/components/ ‚Üí app/components/
```

Most React components work as-is!

---

**Step 3: Convert Pages**

**React:**
```jsx
// src/pages/About.jsx
export default function About() {
  return <div>About page</div>;
}
```

**Next.js:**
```jsx
// app/about/page.tsx
export default function About() {
  return <div>About page</div>;
}
```

---

**Step 4: Update Data Fetching**

**React (useEffect):**
```jsx
function Products() {
  const [products, setProducts] = useState([]);
  
  useEffect(() => {
    fetch('/api/products')
      .then(res => res.json())
      .then(data => setProducts(data));
  }, []);
  
  return <div>{/* render products */}</div>;
}
```

**Next.js (Server Component):**
```tsx
async function Products() {
  const products = await fetch('/api/products')
    .then(res => res.json());
  
  return <div>{/* render products */}</div>;
}
```

---

**Step 5: Add Metadata**

```tsx
export const metadata = {
  title: 'About Us',
  description: 'Learn about our company',
};
```

---

**Step 6: Optimize Images**

**Replace:**
```jsx
<img src="/product.jpg" alt="Product" />
```

**With:**
```jsx
import Image from 'next/image';

<Image 
  src="/product.jpg"

  width={800}
```
